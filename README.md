#### README del Proyecto

**Nombre del proyecto:**
MecaSystem – Sistema de Gestión para Taller Mecánico

**Objetivo:**
Desarrollar una aplicación móvil que permita gestionar de manera eficiente los servicios, clientes, vehículos e inventario de refacciones en un taller mecánico.

**Tecnologías utilizadas:**

* Android Studio (Kotlin/Java)
* Node.js / Express o Firebase
* Firebase Firestore / MySQL
* GitHub
* Jira

#### Manual de Gestión del Proyecto

**Herramientas utilizadas:**

* **Slack:** Comunicación interna del equipo.
* **Google Meet:** Reuniones con cliente y seguimiento.
* **Jira:** Gestión de tareas y control del avance del proyecto.
* **GitHub:** Control de versiones del código.

**Razón de selección:**
Estas herramientas fueron elegidas por su facilidad de uso, integración entre ellas y porque son ampliamente utilizadas en la industria del desarrollo de software, lo que permite simular un entorno real de trabajo.

**Flujo de trabajo:**

1. El analista define requerimientos e historias de usuario.
2. El equipo planifica tareas en Jira (backlog y tablero Kanban).
3. Los desarrolladores trabajan en funcionalidades específicas (frontend y backend).
4. El código se gestiona mediante GitHub usando ramas.
5. QA realiza pruebas y reporta errores.
6. Se realizan reuniones periódicas para seguimiento del proyecto.
7. Se entregan avances al cliente para validación.

Este proceso permite mantener un control adecuado del proyecto, mejorar la comunicación y asegurar la calidad del producto final.
